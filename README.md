## Third Party Vendor Security Assessment

## Overview

This project simulates a third-party security assessment performed by a financial institution evaluating a cloud-based HR platform.

The assessment evaluates the vendor's security posture through an inherent risk assessment, security questionnaire, review of publicly available security documentation, and an executive risk memo.

## Scenario

A financial institution plans to implement a cloud-based HR platform responsible for storing and processing employee information including:

- Social Security Numbers
- Direct Deposit Information
- Payroll Records
- Tax Information
- Employee Contact Information

## Project Objectives

- Assess inherent vendor risk
- Develop a vendor security questionnaire
- Review industry security documentation

## Inherent Risk Assessment

The bank plans on using a third party cloud-based HR tool to store and process highly sensitive employee information, the inherent risk would be high. The vendor would have access to PII and payroll information, potential impacts include identity theft, payroll fraud, and loss of trust within the employees. The vendor being a third party, the bank would have to rely on the vendor's best security practices, the vendor would be a vital part of the business operations and the bank's credibility would be diminished along with financial and legal consequences in a security breach.

## Evidence Reviewed

Since this is a mock security assessment I will be using Google Cloud public information. 

- Google Cloud Trust Center

Google Cloud Trust Center explains why companies trust Google for use of Google Cloud. The trust center breaks down trust into 3 distinct parts that are security, cloud compliance, and data privacy.

- Google Cloud Security Documentation

Google Cloud Security Documentations list the technical controls used to protect customer data. The documentations are broken down into categories involving security operations; access management; audit, monitoring, and logging; cloud governance; and data security etc.

- Google Cloud SOC 2 Overview

Google Cloud SOC 2 compliance regularly undergoes audits based on this standard. They are generated quarterly within a year and accessible through Compliance Reports Manager.

- Google Cloud ISO 27001 Documentation

ISO 27001 is a widely used framework for Information Security Management Systems. It is used to implement, maintain and continually improve systems. Google has earned this certification for many of its systems, processes, and data centers.
